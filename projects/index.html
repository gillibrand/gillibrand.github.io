Global Configuration Management (GCM)

GCM manages versioned artifacts across apps in the IBM Engineering Lifecycle Management (ELM) suite. I wrote and maintained almost all of its web UI, many REST services, and some of the Java backend services.

Tree Component

I wrote GCM's main tree component to support lazy-loading, fast re-renders, animated changes, keyboard navigation, multi-select, drag-and-drop, and more. This completely replaced a 3rd party component that didn't offer the speed or features I needed.

Query Builder

The GCM query builder UI can search based on attributes like dates, tags, numbers, custom enums, and more. It can nest boolean conditions, and subqueries. It's easy to refine complicated queries by previewing results inline and using drag-and-drop to move conditions between groups.

I wrote this UI, plus the Java services to parse its internal domain-specific language and translate that into the actual database queries.

History on Canvas

This history view shows the different branches and baselines of a stream. I used a <canvas> element to to draw the connections. The optional mini-map in the bottom right can pan around large graphs. There are several ways to collapse, filter, and navigate to support thousands of loaded nodes.

DOORS Next

IBM DOORS Next is a market leading requirements management tool that I worked on for a short time. I contributed many bug fixes and serviceability improvements, but I also had the opportunity to refresh its query builder (aka, filters).

Carbon Query Builder

The query builder was an existing feature. I revised it to align with the IBM Carbon Design guidelines, added support for nested AND/OR groups, drag-and-drop reordering, and improved accessibility and usability.

Inline Delete

The old query builder design let you remove conditions with a single click. For the new design I added inline "Remove" confirmations directly on the buttons. My comprise that added a safety-net, but kept the speed of editing customers expected.

Report Builder

Report Builder creates and runs reports on artifacts that span the IBM ELM suite. I bootstrapped the project to replace its jQuery UI with a modern TypeScript/React UI. That included initial project setup with build scripts for dev and production; creating the core APIs to work with existing backend services, including authentication; and building some of its core UI components.

Bulk Tag Editor

I built a React tag component for Report Builder that needed a variation for when multiple reports are selected. I worked closely with a UX designer on this version that looks similar to the single-report version, but uses menus, icons, and color to present the exact changes taking place. There are a lot of possible states, but the most common use cases proved easy-to-use with users.

Pix Grid for Playdate

A pixel art app for Playdate. Written in Lua. Available on GitHub.

URL Clown for Chrome

A Chrome extension for editing URLs. I needed this for work and existing extensions didn't handle hash parameter key-value pairs. Available on Chrome Web Store and Github.
